Introduction¶

Clustering methods are techniques for partitioning a dataset into groups of similar observations. This can be useful for identifying latent categories or communities to which observations in a dataset belong. Although there are many clustering methods (KMeans, Hierarchical Agglomerative, DSCAN, Optic) for researchers to choose from, one thing many of them have in common is that they try to minimize some form of within-cluster variation. For example, K-means clustering, which is the simplest and most popular clustering method, works to minimize the within-cluster sum of squared differences between each observation and the centroid of its cluster.

One problem with clustering methods is that the researcher generally does not know beforehand which variables best describe the latent clusters of the data. Running a clustering method on a dataset with many variables can result in poorly defined clusters, particularly if only a few variables capture the clustering structure of the data and their variances are small compared to the other variables in the data. Standardizing the variables does not solve this problem. Rather, centering the columns and dividing by their standard deviations swaps one set of (equal) weights for another. Specifically, standardization assigns weights to the columns that are inversely proportional to their standard deviations. If the clustering structure of the data is best described by a variable with high variance, standardization will degrade the quality of the clustering method.

I wrote an algorithm to weight a dataset's variables to minimize the within-cluster sum of squared errors that results from running KMeans clustering on the weighted data. Because minimizing the within-cluster sse is the same as maximizing the between-cluster sse, this "Weighted KMeans" algorithm finds a clustering structure in which observations from the different clusters are better separated than clusters from unweighted KMeans. At the end of this notebook, I show that these clusters do weighted KMeans do a better job of differentiating between different categories of observations when "ground truth" classes are available (i.e. the clusters have a higher ARI score).

Algorithm¶

The algorithm for running weighted KMeans is inspired by a paper from Journal of Applied Statistics (Huh and Lim 2009). I wrote it in Python and you can find it here. Here are two explanations of the algorithm, a long one and a short one.

The Long Version¶

To understand how the algorithm works, first consider the idea of a simplex. A n-simplex is just a triangular structure in n dimensions. If p=2, the simplex is a triangle, if p=3 the simplex is a tetrahedron, ect. This construct is useful for modeling mixture components, or weights that sum to 1. Let's say, for instance, we have a dataset with three variables. All possible mixtures of the variables in this dataset could be represented by a 2-simplex, or triangle, where each of the three vertices represents a pure component without the two others (1,0,0 / 0,1,0 / 0,0,1), The center of the triangle represents an equal mixture of each component (.33, .33, .33). This idea can be generalized to a (p-1)-simplix for any number of variables.

So every point in a simplex represents a set of weights. If we performed K-Means using these weights, we would get within-cluster sse for a given set of weig/hts. The goal of the algorithm is to find the set of weights that minimizes the within-cluster sse. The algorithm does this by creating a lattice - a set of points of each vertex of the simplex and the midpoint of every edge of the simplex. These points comprise every "pure mixture", where one variable is given all the weight, and every two-part mixture, where two of the variables are given equal weight and the other variables are given no weight. The algorithm then finds the within-cluster sse from K-Means associated with each point in the lattice. Next, it runs OLS on something called the quadratic canonical function. I won't go into the details of this; the important point is that the design matrix is the set of lattice points and y is the within-cluster sse associated with each lattice point.

Finally, using the coefficients of the quadratic canonical function that were estimated by OLS, the algorithm tries to find the set of weights that minimizes the quadratic canonical function (using the BFGS algorithm from Scipy optimize). K-Means is then run on these weights, and the resulting within-cluster sse is compared to the within-cluster sse predicted by the quadratic canonical function.

The algorithm then uses Cox directions to calculate a reduced simplex, which is a smaller simplex in the same number of dimensions as the original simplex that centers on the "potential optimum" (the set of weights found from the BFGS algorithm). The steps above - using K-Means to get within-cluster sse's for a set of lattice points, fitting the lattice and associated within-cluster sse's to the quadratic canonical function, finding the set of weights that minimizes this function - are repeated on the reduced simplex. The difference between the true within-cluster sse of the potential optimum and the within-cluster sse predicted by the quadratic canonical function get smaller (though not monotonically) after several iterations.

Okay - there's a lot of hand waving here and I skipped over some parts. To learn more about working with mixtures and lattices, check out chapter 12 (Experiments with Mixtures) from the book Response surface methodology: process and product optimization using designed experiments. A lot of the concepts I touched on are explained there.

The Short Version¶

The algorithm represents weights using a simplex, a triangular structure where each point is a set of weights. The algorithm searches the simplex for a set of weights that minimizes the within-cluster sse from K-Means. At each iteration, it creates a smaller simplex around a potential optimum, eventually honing in on a locally optimum set of weights.
